Technical Insights

System Performance & Security

Rate Limiting

Seven AI implements intelligent rate limiting to ensure system stability and prevent abuse:

Team Plan

Higher message processing limits for team deployments (up to 50 messages/second)

Enterprise

Customizable rate limits tailored to organizational needs and usage patterns

Rate limits prevent system overload while maintaining responsive performance for legitimate users.

Allowed Origins

Domain whitelisting ensures secure deployment:

// Example allowed origins configuration
allowed_origins: [
  "https://patientportal.example.com",
  "https://clinic.example.org",
  "https://mobileapp.example.io"
]

This security-first approach prevents unauthorized embedding of Seven AI while maintaining flexibility for approved integrations.

API Architecture

Chat API

Seven AI's HTTP API for backend integrations:

API Request Flow

Client App → Chat API → Botpress Core → Response

Key Characteristics:

  • Designed for backend-to-backend communication
  • Maintains separate conversation threads from other channels
  • Requires explicit installation of Chat Integration

Configuration Options:

Parameter Description Required
Encryption Key For request verification (JWT signing) Optional
Webhook URL For receiving chat events Optional
Webhook Secret For webhook request validation If using webhook
